📊 download_status

Poll the status of in-progress or completed downloads. Use download_status with the IDs returned by download_file.

Typical Status Values

StatusMeaning
in_progressDownload is actively transferring data
completeDownload finished successfully
interruptedDownload was interrupted (network error, etc.)
cancelledUser or agent cancelled the download

Testing Workflow

  1. Call download_file with a large file URL (e.g., the NASA image above)
  2. Immediately call download_status with the returned ID — expect in_progress
  3. Wait a few seconds, then call download_status again — expect complete
  4. Call download_status with multiple IDs at once to batch-check statuses

Edge Cases

  • Unknown ID: Call download_status with a non-existent ID — should return an error
  • Empty array: Call download_status with ids: [] — should return empty results
  • Batch: Trigger 3 downloads, then check all 3 statuses in a single download_status call

Cross-Origin — Public API (CORS allowed)

risk
https://httpbin.org/json

httpbin.org sends permissive CORS headers. fetch_resource should succeed and return JSON data.

Cross-Origin — CORS Blocked

high-risk
https://example.com

example.com does not send CORS headers. In Chrome, fetch_resource should still succeed (extension has host permissions). In Safari, it should return cors_blocked status honestly.

Cross-Origin — 404 Not Found

risk
https://httpbin.org/status/404

Returns HTTP 404. fetch_resource should report this as an error, not silently fail.

Large Content — Truncation

risk
https://httpbin.org/html

A moderately-sized HTML page. Test with a small maxSizeBytes (e.g., 500) to verify truncation works correctly and the agent receives a partial result with a truncation indicator.

Fetch Timeout

risk
https://httpbin.org/delay/10

This endpoint delays 10 seconds before responding. Set fetchTimeout to 3 seconds (3000 ms) to test that the tool correctly times out and returns a timeout error rather than hanging.

ℹ️ About P1.7

P1.7 introduces download awareness for AI agents via Brijio. Agents can now trigger browser downloads, poll their progress, and fetch web resource content — all through the MCP protocol.

fetch_resource is classified high-risk because it returns page content directly to the agent. In Safari, CORS-blocked fetches will be reported honestly as cors_blocked rather than silently failing or working around the restriction.
