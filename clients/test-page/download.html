Downloads

Use this page to exercise Brijio download and fetch tools with stable local fixtures first. Remote examples are labeled as intentional edge cases.

Happy paths are same-origin.

The three primary URLs below are served by `brijio demo`, so they should not break because a public placeholder service changed or disappeared.

Expected outcomes

Use this table as a screenshot-friendly checklist while validating the MCP tool responses.

Scenario Tool Expected result
Local text download download_file Download starts and `download_status` reaches complete.
Local JSON fetch fetch_resource Response body contains `same-origin fetch and download fixture`.
Missing local file download_file Failure is reported clearly rather than hanging.
Remote CORS blocked fetch_resource Fetch reports a CORS/network error from browser context.

Tool order

  1. Call `download_file` with one local fixture URL.
  2. Capture the returned download ID.
  3. Call `download_status` until the item completes or fails.
  4. Call `fetch_resource` for the local JSON fixture.
