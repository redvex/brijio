     1|# Security Policy
     2|
     3|## Supported Versions
     4|
     5|BrowserBridge is under active development.
     6|
     7|Security fixes are applied to the latest supported release.
     8|
     9|## Reporting a Vulnerability
    10|
    11|Please do not disclose security vulnerabilities publicly.
    12|
    13|Instead, report vulnerabilities privately to:
    14|
    15|security@browserbridge.dev
    16|
    17|Include:
    18|
    19|- Description of the issue
    20|- Reproduction steps
    21|- Potential impact
    22|- Suggested mitigation (if known)
    23|
    24|## Response Process
    25|
    26|The maintainers will:
    27|
    28|1. Acknowledge receipt
    29|2. Validate the report
    30|3. Develop a fix
    31|4. Coordinate disclosure
    32|
    33|## Security Philosophy
    34|
    35|BrowserBridge is designed around:
    36|
    37|- User-controlled infrastructure
    38|- End-to-end encryption where possible
    39|- Least-privilege permissions
    40|- Transparent operation
    41|- Open review of source code
    42|
    43|## Scope
    44|
    45|Security reports covering:
    46|
    47|- Browser extensions
    48|- WebSocket servers
    49|- MCP servers
    50|- Authentication mechanisms
    51|- Encryption implementations
    52|
    53|are considered in scope.
