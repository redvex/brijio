# Security Policy

## Supported Versions

Brijio is under active development.

Security fixes are applied to the latest supported release.

## Reporting a Vulnerability

Please do not disclose security vulnerabilities publicly.

Instead, report vulnerabilities privately to:

redvex@me.com

Include:

- Description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation (if known)

## Response Process

The maintainers will:

1. Acknowledge receipt
2. Validate the report
3. Develop a fix
4. Coordinate disclosure

## Security Philosophy

Brijio is designed around:

- User-controlled infrastructure
- End-to-end encryption where possible
- Least-privilege permissions
- Transparent operation
- Open review of source code

## Scope

Security reports covering:

- Browser extensions
- WebSocket servers
- MCP servers
- Authentication mechanisms
- Encryption implementations

are considered in scope.
